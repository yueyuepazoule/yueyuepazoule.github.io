PathMeasure

实现Path路径点的坐标追踪的API，类似一个计算器，可以计算出指定路径的一些信息，比如路径总长、指定长度所对应的坐标点等，通过它可以实现复杂的动画效果。

forceClosed参数：表示Path最终是否需要闭合，若为true，无论绑定的Path是否闭合，PathMeasure测量时都会将其会看作闭合（包含闭合的那段路径），但对绑定的Path没有任何影响。

简单方法使用

  • getLength()

    获取计算的路径长度

  • isClosed()

    判断forceClosed参数，此参数若为true则返回true。

  • getSegment()

    startD、stopD参数：控制截取长度

    dst：将截取后的Path添加到参数dst中，而不是替换。

    startWithMoveTo：起始点是否用moveTo将路径的新起始点移到结果Path的起始点，通常设为true，以保证每次截取的Path都是完整正常的，若设为false，则新增的片段会从上一次Path的终点来是计算，保证是连续的。

    实例：支付宝画圈

  • nextContour()

    Path可以由多条曲线构成，getLength()和getSegment()等函数都只会对其中一条线段进行计算。

    而nextContour()可以用于跳转到下一条曲线，跳转成功返回true，失败返回false,跳转顺序与path的添加顺序相同。

  • getPosTan()

    用于得出路径上某一长度的位置以及该位置的正切

    diatance：距离path起始点的长度

    pos：该点在画布上的坐标值，x = pos[0], y = pos[1]

    tan:正切值 x = tan[0], y = tan[1], tan = y/x

    pos[]和tan[]在使用前要分配空间，getPosTan()只对其进行赋值

    知道tan[]后，在Math类中，有两个可以求反正切值的方法,可得夹角度数（弧度值）。

View Animator 视图动画

Android中视图动画由五种类型组成：alpha scale translate rotate set

配置xml动画文件

配置文件时需要以下标签，同时文件存放在res/drawable或者res/anim文件夹下

  • alpha：渐变透明度
  • scale ：渐变尺寸伸缩
  • translate：画面变换位置移动
  • rotate：画面转移旋转
  • set：定义动画集

set标签自身没有属性，它的属性都是从Animation类继承而来。这些属性用于set标签时，会对set标签下的所有子控件产生作用。

ValueAnimator 属性动画

只针对控件的某一属性来做动画，单独改变控件某一属性的值

单击区域问题

属性动画中，手指按下时，触摸事件会经过ViewGroup中的dispatchTouchEvent方法筛选符合条件(手指在边界范围内)的子View进行分派事件。它会先调用子View的hasIdentityMatrix方法来判断这个View是否应用过位移、缩放、旋转之类的属性动画，然后会将触摸的坐标通过逆变换变回去。

补间动画中，在draw的时候，会检测有无Animation,如果有则将动画的变化应用到Canvas上，再画出来。
